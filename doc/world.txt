Title: Kirikiri/KAGEX World Extension Functions

Author: Wamsoft L.P. Tsuyoshi Watanabe



● KAGEX World Extension Functions Overview



By loading `world.ks` as a plugin into KAGEX, you can use the special "World Functions".



In the World Extension, elements that make up the game are considered "Objects,"
and the scripting style involves performing operations on these objects.



Environment : Control of the entire system and camera.
Stage       : A dedicated object for controlling the location and the "Background Image" associated with it.
Event       : A dedicated object for controlling "Event Images" (CGs).
Character   : An object for controlling "Character Images" (standing sprites), voices, etc.
Layer       : An object for controlling "Layer Images" as screen effect materials.



All objects share the following characteristics:



(1) KAG "Attribute" names are considered commands.
(2) KAG "Attribute Values" are considered parameters for those commands.



● Regarding Major Specification Changes



・`hide` type commands have been abolished. Please use `all` type commands + `hide` instead.
・`clear` command has been abolished. Please use `all` type commands + `hide` instead.
・`newchar`/`delchar`, `newlayer`/`dellayer` commands remain but are deprecated.
  Please use `char` / `layer` for creation, and use the `delete` command of the respective object for deletion.



● Installation Method



It is installed automatically by default. If you do not use it,
please set `global.worldEnabled` to `false` in `startup.tjs` or similar.



● Extended Tags



◇ Extension of Basic Processing



KAGEX tag processing is extended as follows:



・The `ENV` tag becomes a command for the "Environment".
・The `STAGE` or `BG` tag becomes a command for the "Stage".
・The `EVENT` or `EV` tag becomes a command for the "Event".
・The `CHAR` tag, combined with the `name` option, becomes a command for a "Character".
・The `LAYER` tag, combined with the `name` option, becomes a command for a "Layer".
・If there is a location specification, time specification, or action specification command matching the tag name,
　it becomes a command for the "Stage".
・If a character matching the tag name exists, it becomes a command for that character.
・If a layer matching the tag name exists, it becomes a command for that layer.
・"Environment" commands matching the tag name function as is.
・Name processing functions are processed in synchronization with character voices.



・The `BGM` tag becomes a command for the "BGM Object".



  ※ Specifications starting with "bgm" are treated as filenames,
　   equivalent to `[BGM play=Name]`.



・The `SE` tag becomes a command for the "SE Object Group".



  ※ If you specify a name starting with "se", it is considered a specification
　   for the SE currently playing with that name.
　   If no SE is playing, it is treated as equivalent to `[SE play=Name]`.



・Release conditions for delayed execution (`delayrun`) are added.



　String specification also reacts to labels within voice files.
　If voice playback does not occur, the string specification forces execution at the beginning.
  Labels within voices function with their original names and also with aliases 
　`vl1`, `vl2`, `vl3`... in order of appearance.
  Additionally, the virtual label `vlstop` is processed when the voice ends.



◇ Automatic Flag Processing



・When an image starting with "EV" is drawn on the Event or Stage,
　a value "cg_Name" is automatically registered in `sflags`.



・When BGM is played using the `BGM` tag,
　a value "bgm_Name" is automatically registered in `sflags`.



・When a character voice is played, a flag "voice_VoiceDefinitionName"
　is automatically registered in `sflags`.





・When an image is drawn on a layer,
　a value "cg_Name" is automatically registered in `sflags`.



・When a tile image is drawn on a layer,
　a value "tile_Name" is automatically registered in `sflags`.



・When a GFX type effect is displayed on a layer,
  a value "autoflagDefinitionName_Name" is automatically registered in `sflags`.



◇ Command Extensions



◆ begintrans / endtrans



Executes a transition for the entire screen.



・When `begintrans` is executed, the content of the front screen is copied to the back screen.
・From this point until the `endtrans` command is executed, all drawing for environment-related objects
　is performed on the back screen.
・When `endtrans` is executed, a transition for the entire screen is executed with the specified parameters.



Attributes of begintrans
    trans   Transition name specification (*Details described later)
    fade    Fade specification shortcut (*Details described later)
	nosync	Executes the global transition asynchronously



 ※ `ltbegin`/`ltend` are adjusted to function similarly.



◆ msgoff



Erases the message window.



Attributes
    trans   Transition name specification
    fade    Fade specification
    nofade  Suppress automatic fade



If neither `trans` nor `fade` is specified,
standard fade processing using opacity is performed instead of back/front screen transition.
※ The fade time in this case can be specified with `kag.messageeFadeTime`.



◆ msgon



Displays the message window.



Attributes
    trans   Transition name specification
    fade    Fade specification



Like `msgoff`, if neither `trans` nor `fade` is specified, standard fade processing is used.



◆ layer



Functions as a command for layer objects.
Can be used when you want to specify a layer name as a parameter in a macro, etc.
If a non-existent layer is specified, it is automatically created.



Attributes
    name    The identifier name of the layer.



◆ char



Functions as a command for environment character objects.
Can be used when you want to specify a character name as a parameter in a macro, etc.
If a non-existent name is specified, it is automatically created.



Attributes
    name        Specifies the character name.
    initname    Initialization name.



◆ allse



Executes the same command for all SEs.



◆ showvoiceinfo



◆ twait or *



Waits for a time from the start of the message display or for a label within a voice.
Basically used in combination with `delayrun` specification.



Attributes
	name	Voice label specification
	time	Wait time specification



If attributes are unspecified, it waits for automatic label specifications `vl1`, `vl2`, `vl3`, etc.,
in the order they appear in the text.



Example: Just insert `[*]` at the timing where the label appears in the text.



[Akari Surprise]
[Akari Startled delayrun=vl1]
[Akari Cry delayrun=vl2]
【Akari】Eh, [*] wha, [*] no way~

◇ All-type Commands



When you want to process image-type objects in a batch, you can use descriptions starting with `all~`.
These commands basically mean the same as listing the same command one by one for objects
displayed at that time.
In other words, if transitions or similar are synchronized, they are executed one by one in order.
If you want to make them all operate simultaneously:



a. Add `nosync` specification to make them all operate asynchronously.
b. Add `all` specification to perform a global transition.



Please do one of the above.



※ Default values when these are unspecified can be defined in `envinit.tjs`.
allnosync : Default is considered `nosync` specification.
allall    : Default is considered `all` specification.



Common Attributes
	poscond Limit the display position if the target is a character (specify the same as pos specification).
	vcond	Limit the display state of the target.
			true: Those currently displayed (Default)
			false: Those hidden
			all: All
	all		Executes while enclosing the whole operation in `begintrans`/`endtrans`.



※ Example
　Discard all hidden layers
  [alllayer delete vcond=false]



◆ allchar	Executes the same command for all displayed characters.
◆ alllayer	Executes the same command for all displayed layers.
◆ allfore	Executes the same command for all displayed characters/layers.
◆ allbase	Executes the same command for the event image/stage.
◆ allimage	Executes the same command for all displayed image objects.



● Omission of Description in World Extension



Various shorthand notations are valid in World Extension.



◇ Omission of Environment Attribute Specification



-------------------------
[stage stage=School]
-------------------------



This is the notation to set the stage to "School".
If the registered name does not conflict with other specifications, it can be omitted as follows:



-------------------------
[stage School]
-------------------------



◇ Omission of Environment Name



If the attribute value does not conflict with other tags, it can be further omitted as follows:



-------------------------
[School]
-------------------------



Since the tag "School" does not exist, it is replaced with a command for the stage
and executed, becoming equivalent to `[stage School]`.



◇ Character Specification



-------------------------
[Minase dress=Uniform pos=Middle]
-------------------------



Since neither the tag "Minase" nor a command for the environment exists,
it is processed as a command for the character "Minase".
※ The character must be registered in `envinit.tjs` in advance.



Layers created with `newlay` behave similarly.



◇ Omission of Character Attributes



-------------------------
[Minase Uniform Middle]
-------------------------



Like the environment, if the name does not overlap with others,
the parameter name specification can be omitted. The same applies to layers.



◇ Character Voice Playback



---------------------------------------------
【Tae/Voice】「Wake up... hey, wake up I said...」
---------------------------------------------



・Plays the default voice for character "Tae" at that point.
・Displays the name as "Voice" in the message window.
・Automatically displays the face column of the message window with "Tae's" current expression + pose.







    voice       Specifies the voice to play next.
　　　           Number:   Specifies the playback numeric counter.
        　　　   String:   Interrupts with a one-time playback.
            　　 ignore:   Ignores playback only once (treated as a string).



　  once        Interrupts for one-time playback using a number specification (treated as a string).
　  incvoice    Increments the numeric counter even when a string is specified.
    noincvoice  Does not increment the numeric counter (stronger specification than incvoice).



    clearvoice  Stops voice playback from the next time (returns to initial state).
    nextvoice   If specified, plays voice simultaneously with the next voice playback of another character.

        Normal notation:  data = to

　　　・For coordinates, it's the edge of the screen.
　　　・If originally a % specification, it remains so.
　　　・For rotation, it's 360 degrees.

◇ Voice Specification



If there is a description of 【Name】 at the beginning of the line, the voice is automatically played.



Basic Operation
　・If a number is specified, the voice follows the format and plays,
　　and the numeric counter is incremented.
　・If a string is specified, it plays taking priority over the numeric specification,
　　and the numeric counter is not changed.
　・String specification is valid only once.



◆ KAG Commands



    playvoice   Directly plays the specified voice.
                When specifying a voice name, it does not affect the voice count.
                When unspecified, it operates by referring to the internal count,
                and the count is incremented.
    
    waitvoice   Waits for the currently playing voice to finish.
                If set to `true` (default), cancellation by click is valid.
                If set to `false`, clicks are suppressed.



    stopvoice   Forcibly stops voice playback.



    bvoice      Specifies a voice to be loop-played in the background.
                Stops while other voices of the corresponding character are playing. When that voice stops,
                it resumes and loop plays. Stops if an empty string is specified.
	Auxiliary Parameter
	loop	Specifies looping.



Example: Play by specifying voice number
------------------------------------
[Tae voice=1000]
【Tae】Good morning
------------------------------------



Example: Play by specifying voice file
------------------------------------
[Tae voice="tae1000"]
【Tae】Good morning
------------------------------------



Example: Remove voice playback from next time
------------------------------------
[Tae clearvoice]
------------------------------------



Example: Handling a case where one voice is missing
------------------------------------
[Tae voice=1000 once]
【Tae】Good morning
------------------------------------



Example: Handling a case where one voice is replaced
------------------------------------
[Tae voice=1000 once incvoice]
【Tae】Good morning
------------------------------------



Example: Handling a case where silence is inserted (count continues)
------------------------------------
[Tae voice=ignore]
【Tae】......
------------------------------------



Example: Handling a case where voice is removed for one line (discard corresponding count)
------------------------------------
[Tae voice=ignore incvoice]
【Tae】......
------------------------------------



・In the environment initialization process, voices for all characters are initialized to "None".
・The actual file name format for number specification is defined in `envinit.tjs`.



※ About Wait Time during Voice Specification



When voice playback occurs,
the wait time at page wait during auto-play changes as follows:



Normal
　Waits for the specified time.
During Voice Playback
　Remaining voice time at the moment the wait started (calculated from voice length) + specified time wait.

Clicking cancels the wait and proceeds to the next conversation.

● About Actions



Coordinates, magnification, etc. of a layer can be changed over a specified time.
This function is called an "Action".



◇ Automatic Action



If `time` is specified simultaneously with parameters like `zoom`, `rotate`,
those parameters are automatically action-processed.
A smooth change from the current value to the specified value is executed over the specified time.
However, if `trans` is specified, transition processing takes priority.



Example: Magnify to 2x size over 1 second.
[layer zoom=200 time=1000]



Note that when specifying parameters, if you use initialization specification (Parameter=SpecifiedValue:InitialValue),
it first resets to the initial value and then executes the action until it reaches the specified value.



Acceleration/Deceleration of the action can be specified with the `accel` parameter.



◇ Action Command



Advanced actions can be applied by defining actions.
Operations on multiple parameters can be described in these definitions.



1. Define name and parameters in `actions` of `envinit.tjs`, or
　 create a definition file in the form `action_Name.tjs`.



2. Specify with `action` parameter.



※ Default parameters can be overwritten by other parameter specifications.



◇ Action Synchronization Rules



Principle 1: Action specifications are executed asynchronously.



Scripts proceed even while an action is in progress.
If waiting for processing until the action completes, specify the `sync` parameter.



Actions operating synchronously with `sync` specification are canceled by click operations.
Cancelled actions enter the state where the specified action is completed.



Principle 2: Actions in operation are canceled by page break processing.



If you want to ignore page break cancellation and continue the action
across multiple texts, specify the `nowait` parameter.



Note that using `stopaction` forces actions specified with `nowait` to stop as well.



● About Transitions



The process of switching the display content of a layer is called a "Transition".



Transition processing is executed in the following cases:
・When displayed
・When erased
・When display content changes



◇ Individual Transition and Global Transition



In World Extension, in principle, transitions can be specified for the display of individual objects.
However, in the following cases, it is automatically treated as a Global Transition:



・When the display level of an object changes.



・When commands affecting other objects are used.



  - Changing background with background effects (Effect layer changes).
　- Changing background time (Character changes simultaneously).
  - Changing background while Event CG exists (Event CG is erased).
  - Executing environment commands for effect manipulation (Effect reaches all images).



・When a transition specified with the `trans.transall` parameter is used.



A Global Transition is a process that transitions the entire screen at once.
Besides the aforementioned automatic processing, it can be explicitly executed with `begintrans`/`endtrans`.



◇ Specification Method



1. Define name and parameters in `transitions` of `envinit.tjs`, or
　 create a definition file in the form `trans_Name.tjs`.



2. Specify with `trans` parameter.



※ Default parameters can be overwritten by other parameter specifications.



◇ Transition Synchronization



Transitions wait for processing to complete by default (sync state).
However, if `nosync` is specified, it executes asynchronously.
If a transition is executed on a layer currently executing a transition,
it executes after the previous one is cancelled.



This behavior can be reversed by defining "nosynctrans" in `envinit.tjs`
(Default becomes asynchronous like action, sync can be done with `sync`).
Also, transitions specified with the `waittrans` parameter are synchronized.



Transitions operating synchronously are canceled by click operations, just like actions.
Cancelled transitions enter the state where the specified transition is completed.



Transitions in operation are canceled by page break processing, just like actions.
If you want to ignore page break cancellation and continue the transition
across multiple texts, specify the `nowait` parameter.



Using `stoptrans` forces actions specified with `nowait` to stop as well.



Global Transitions can also be executed asynchronously with `nosync` specification like individual transitions,
but in principle, only one Global Transition can be executed at a time.
During a Global Transition, if another Global Transition or drawing update processing (various transitions and actions)
of individual objects is executed, the currently running Global Transition is cancelled
to prevent inconsistencies.



※ A warning message will be displayed in the log.
　If this appears, the timing specification is unreasonable, so
　it is recommended to review the description.



● delayrun



Commands can be executed with delay.



[Command delayrun=DelaySpecification]



・Delay is canceled before page break processing by lineMode (after click wait),
  or by the `[delaydone]` command.
・If a number is specified, it executes after delaying for the specified time (in ms).
・If a string is specified, it executes at the timing of the label in the voice played next.
・With string specification, if there is no voice playback, execution is forced at the beginning.
